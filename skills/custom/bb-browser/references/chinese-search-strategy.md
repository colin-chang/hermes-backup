# 中文搜索策略：搜索引擎选择指南

## 核心结论

**搜索中文品牌名/产品对比/消费评测时，Baidu 是唯一可靠的选择。Bing 和 Google 均存在严重的中文分词问题。**

---

## 各搜索引擎实测效果（2025-06）

### Baidu（`bb-browser site baidu/search`）✅ 推荐

- **中文品牌名处理**：正确识别"蕉内"、"蕉下"等复合品牌名
- **产品对比查询**：能搜到知乎/百度知道/什么值得买上的型号对比文章
- **长尾查询**：对"500A 301S 区别 阿罗裤"这类复合查询返回精准结果
- **缺点**：结果中包含推广链接，需人工过滤

### Bing（`bb-browser site bing/search`）❌ 不可用

- **严重分词错误**：将"蕉内"强制拆分为"蕉"（banana），搜索结果全是香蕉/芭蕉/汉字字典
- **引号不生效**：`"蕉内"` 加引号后仍被忽略
- **结论**：对中文品牌名完全不可用，不要尝试

### Google（`bb-browser site google/search`）❌ 不可用

- **结果乱码/不相关**：返回 1996 年游戏杂志文本、品牌排行榜等完全无关内容
- **可能是 adapter 问题**：Google adapter 对中文查询的处理存在 bug
- **结论**：中文产品查询走 Google adapter 纯属浪费时间

### 其他搜索引擎

| 引擎 | 状态 | 说明 |
|------|------|------|
| DuckDuckGo | ⚠️ 未经测试 | 可能可用，但 adapter 覆盖率有限 |
| 搜狗微信 | ⚠️ 场景局限 | 仅搜微信公众号内容，不适合商品对比 |

## 推荐搜索流程

```
中文消费品牌/产品对比查询？
├── 1. bb-browser site baidu/search "品牌 型号1 型号2 区别" --json
│     → 从 snippet 中判断是否有对比文章
├── 2. 需要深入？→ 点进百度知道/知乎/什么值得买的具体页面
│     → 需要登录的平台（知乎/什么值得买）可能被拦截
└── 3. 若 Baidu 返回的页面需要登录/JS 渲染
      → 从 snippet 中提取信息拼凑答案（如本次会话所做）
```

## 已知不可用平台（中文消费查询场景）

| 平台 | 失败原因 |
|------|----------|
| 什么值得买 (smzdm) | Anti-bot probe 拦截（`TypeError: Failed to fetch`） |
| 知乎 (zhihu) | 需要登录（HTTP 401） |
| 京东 (jd.com) | 需要 JS 渲染 + 可能需登录 |
| 天猫 (tmall.com) | 强制跳转登录页 |
| 小红书 | adapter 不存在，详情页 SPA 需 JS 渲染 |

## 经验教训

1. **不要用 Bing 搜中文品牌**：分词 bug 会导致结果完全跑偏，浪费多轮尝试
2. **Google adapter 对中文查询不可靠**：不要指望它能搜到中文产品信息
3. **Baidu 是唯一出路**：虽然结果质量参差，但至少能搜到相关内容
4. **接受不完美**：中文消费内容生态封闭（登录墙、反爬），很多页面无法直接抓取全文。从搜索 snippet 拼凑信息是务实策略
