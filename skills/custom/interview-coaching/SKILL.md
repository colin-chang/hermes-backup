# 行为面试准备教练

当用户（或用户的家人朋友）面临终面行为面试（Behavioral Interview）时使用本技能。适用于 Financial Analyst、Software Engineer、Product Manager 等岗位的行为规范/文化适配面试。

## 触发条件

- 用户说「帮我准备面试」「最后一轮面试」「行为面试」「behavioral interview」
- 用户提到具体公司 + 岗位的终面准备
- 用户需要 STAR 故事打磨或模拟面试

> ⚠️ **触发后第一步：确认时间线**。不要假设「今晚到明天」还是「几天后」——直接用 `clarify` 工具问清楚面试还有多少时间。这决定了用标准模式（3 天）还是紧急模式（≤3 小时）。

## 方法论

### Phase 1：公司/岗位雷达（先摸清对手）

> ⚠️ **工具选择铁律**：公司调研必须遵循 `tool-selection-strategy` 的优先级链——`web_search` 发现 URL → `dokobot read --local` 抓取全文。**禁止使用 curl/浏览器工具做信息获取**（那是浏览器自动化路径，不是只读信息获取路径）。

1. **公司业务结构拆解**：列出公司的主要业务板块，标注各板块与目标岗位的关联
2. **战略优先级抓取**：从官网 About Us / 年报 / 新闻中提取公司的使命、核心价值观、当前战略方向
3. **岗位面经搜集**：Glassdoor、RedFlagDeals、Reddit、Medium 等平台搜索「公司+岗位+interview」，提取面试结构（几轮/几道题/时长）和真实问题
4. **公司文化关键词提取**：从 Code of Conduct / ESG 报告 / 领英公司页面提取高频价值观词，用于定制回答中的文化对齐

### Phase 2：行为问题预测

按高/中/低三档概率排列可能的问题，每档 2-3 题：

| 等级 | 特征 | 示例 |
|------|------|------|
| 🔴 P0 必问 | 与岗位核心能力直接挂钩 | 数据分析沟通、错误处理、deadline 管理 |
| 🟡 P1 高概率 | 与团队协作/公司文化相关 | 分歧解决、流程改进、动机问题 |
| 🟢 P2 中概率 | 情境型/补充型 | 快速学习、模糊情境处理 |

**问题设计原则**：
- 不套用通用面试题库，必须结合**该公司业务场景**定制
- 每个问题标注「真正在考察什么」（表层问题 vs 深层意图）
- 指出常见翻车点和正确打开方式

### Phase 3：STAR 故事定制

针对候选人特点（优势/弱点），提供：

1. **回答模板**：每个高频问题给出完整的 STAR 结构示例
2. **「三句话压缩训练」**：把每个 STAR 故事先压缩成三句话（S+T → A → R），练到脱稿流利后再扩展
3. **「关键词链记忆法」（紧急模式首选）**：每道题只记 4 个关键词（如 scope → prioritize → communicate → deliver），面试时用这 4 个词串成句子。比背诵全文容易得多，且听起来自然。
4. **追问轮预案**：预测面试官可能追问的 2-3 个方向，准备应对思路
5. **弱点翻转策略**：帮候选人把真实弱点包装成「已意识 + 已行动」的成长故事

### Phase 4：练习计划

**模式 A：标准冲刺（3 天）**

| 天 | 任务 | 方法 |
|----|------|------|
| Day 1 | 故事写作 | 写出 5 个万能故事的核心版本（80-100 字），再口头扩展 2 分钟版。录音回听。 |
| Day 2 | 追问模拟 | 找人扮演面试官，对每个故事追问 2 轮（决策理由/时间压力/同事反应） |
| Day 3 | 全真模拟 | 穿正式服装，完整跑 45 分钟面试。控制语速（中文母语者讲英文易加速） |

**模式 B：紧急模式（≤3 小时）**

> ⚠️ 必须先确认面试时间！不要假设「今晚到明天」——用户说「几个小时之后」就是字面意义上的几小时后。

| 时间段 | 任务 | 方法 |
|--------|------|------|
| 第 1 小时 | **只写最核心的 3 个故事**（P0 必问题） | 每题 5 句话以内，R 必须有数字。不要写 5 个——来不及也没必要。 |
| 第 2 小时 | **口头讲，不写** | 每个故事对着镜子/手机录视频讲 3 遍。第一遍卡壳正常，第三遍会顺。同时过一遍公司价值观关键词。 |
| 第 3 小时 | **休息 + 放空** | 不要再塞新内容——焦虑会把已记住的也挤出去。洗个脸、深呼吸。 |
| 面试前 15min | 默念关键词链 | 脑子里只过 4 个关键词，不过全文。喝水（紧张口干是真实风险）。 |
| 面试中 | 卡壳时用救生圈句 | "That's a great question — let me think of a specific example." 给自己 5 秒。 |

### Phase 5：面试日战术

- **弱点的表述**：永远不说「我不会」，说「这方面我目前经验有限，但我的理解是…」
- **提问环节**：问岗位痛点和团队挑战，别问休假福利
- **语速红线**：讲完一句话默数 1-2-3 再讲下一句

## 加拿大职场特化注意

加拿大企业行为面试和美国有显著差异：

- **Consensus > Confrontation**：分歧故事必须体现「尊重对方立场 + 数据说话找到共识」
- **Process > Heroics**：加班故事不加分——他们想听的是你设计了更好的流程，不是你能熬夜
- **Self-awareness > Perfection**：暴露真实弱点 + 证明你在改，比编造「我太完美主义」高级得多

## 输出格式

```
1. 公司/岗位分析（业务结构 + 战略优先级 + 岗位定位）
2. 预测问题清单（P0/P1/P2 分级，每个问题附考察意图 + 翻车点 + 应答策略）
3. 万能故事模板（5 个 STAR 故事 + 对应问题映射表）
4. 练习计划（分天任务 + 模拟脚本）
5. 面试日检查清单
```
