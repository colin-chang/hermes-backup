# 英语学习计划设计

> 为 Colin 设计英语学习方案。Colin 被动词汇 ~12K / 主动词汇 ~6-7K / 美音敏感 / 目标：多伦多生活英语。

## 用户英语水平画像

| 维度 | 现状 | 核心瓶颈 |
|------|------|---------|
| 词汇 | 被动 ~12K / 主动 ~6-7K | 被动词汇 → 主动输出的激活通道未打通 |
| 听力 | 美音相对敏感 / 英音困难 | 音系通路解码速度慢（视觉认识但听觉检索失败） |
| 口语 | 简单句流畅标准 / 复杂句卡顿 | 句法生成速度跟不上思维，口语流畅度不足 |
| 发音 | 单音标准 / 语流中退化 | 6-7 年未开口，口腔肌肉程序需要重建 |
| 阅读 | 能认识大量词汇 | 阅读速度慢 |
| 写作 | 未正式评估 | 预计拼写和句法结构需要训练 |

## 核心设计原则

### 1. 材料选择铁律

- **英音 vs 美音：** 目标为多伦多生活 → 前 10 周完全使用北美英语材料。英音仅在最后 2 周做适应性暴露（每周 2 次 × 20 分钟）。
- **加拿大口音优先：** Bob the Canadian 是唯一的加拿大本土教师频道，语速放慢、户外实景、B1 难度——优先使用，作为「从学习材料到真实英语」的中间层。
- **内容 > 语速：** 泛听材料的内容必须与真实生活场景相关（租房/看诊/银行/社交），而非全球政治新闻。All Ears English 和 Life Kit 的内容实用性远高于 NPR Up First。

### 2. 听力瓶颈的神经机制理解

用户的核心问题是「阅读认识但听不出」——这不是词汇问题，是**音系通路解码速度不足**。

双通路模型：
- **字形通路（已充分髓鞘化）：** 眼睛看到文字 → 0.1 秒 → 理解
- **音系通路（髓鞘化不足）：** 耳朵听到声音 → 需切分连续语流 → 匹配失败 → 丢失

**突破策略：不是逐个单词去练，而是训练音系处理器本身。** 三个并行方法：
1. 连读/吞音边界训练（利用阿飞口语 Day 16-31）
2. 快速听觉词汇反应训练（Anki「听力盲区」Deck 听音辨义，限时 2-3 秒）
3. 影子跟读（0.5-1 词延迟实时跟读，不暂停）

### 3. 学习工具选择

- **Anki（首选 SRS 工具）：** 开源、本地 SQLite 存储、离线可用、macOS 免费。用于间隔重复（SM-2 算法）。
- **欧路词典（纯词典查询）：** 网络不稳定/Bug 多/国外几乎不可用。仅保留作为查词工具，不做背单词。
- **豆包（字节跳动，口语对话首选）：** 国内直连流畅、速度快、语音效果好，免费可用。英文对话能力足以覆盖日常场景训练。每周日用 System Prompt 设定当周 5 个场景。如果豆包语音模式不支持 System Prompt，则每次对话开始时用英文发一句话设定场景。备用：ChatGPT（附录 E 同时提供两套 Prompt）。

### 4. 播客评估方法

在推荐泛听播客前，必须先核实：

| 维度 | 检查点 |
|------|--------|
| 平台功能 | Apple 播客可调 0.5x-2x 速，有内置 transcript |
| 语速 | 是否为学习者的「合适难度」——太慢 = 无训练效果，太快 = 挫败 |
| 内容匹配 | 话题是否与用户的实际使用场景对齐 |
| transcript | 是否免费可用（Apple 播客内置或官网提供）|

**已评估播客速查：**

| 播客 | 语速 | 内容 | 适用阶段 |
|------|------|------|---------|
| All Ears English | 母语自然（可降速） | 日常会话表达、惯用语 | Phase 1-4 泛听主力 |
| Life Kit (NPR) | ~150 WPM（清晰） | 生活技能话题 | Phase 2-4 |
| NPR Up First | ~150-200 WPM | 全球新闻 | Phase 3+（内容不匹配，降级使用） |
| CBC News: World Report | ~160 WPM | 加拿大新闻 | Phase 4 |

## 四时段学习模板

> 设计原理：高认知负荷模块放上午精力高峰，中等负荷放午后，互动型放晚间。

| 时段 | 时间 | 学习量 | 模块 |
|------|------|:---:|------|
| 🌅 上午 | 06:30-09:30 | 2h 20min | 精听(50min) + 阿飞口语(45min) + ChatGPT口语(30min) |
| 🍽️ 午休 | 12:30-14:00 | 35min | 午餐泛听(20min) + Anki(15min) |
| 🌤️ 下午 | 17:00-18:00 | 50min | 阅读 |
| 🌙 晚间 | 19:00-22:30 | 2h 30min | 口语实战(40min) + 写作(40min) + HIMYM/场景(30min) |

### 模板通用性

⚠️ **时间框架在四个阶段完全通用**——变的不是时间，是各模块内填充的**具体材料**和**难度**。精听永远是 07:00-07:50，但材料从 Phase 1 的 Bob the Canadian 逐步升级到 Phase 4 的 CBC 直播。就像健身房固定器械——动作模式不变，加片就行。在计划文档中，§3 仅列出每天的「内容」列，§1.2 的时间表不需要逐阶段复制。

## 三层精听法

代替传统听写（dictation），用**口头复述**代替「写」，让训练始终保持在听说通路上。

> ⚠️ **L1/L2/L3 使用同一段音频。** 三层不是三个不同的材料，而是对同一段材料的三层递进式「用法」——L1 手术刀级逐句解剖（30min）→ L2 段落级信息抓取（15min）→ L3 全文级真实对话模拟（5min）。总计正好 50 分钟，每一层训练的是不同的听力能力。不要给每层准备不同材料。

| 层级 | 方法 | 目的 | 时间 | 最高遍数 |
|------|------|------|:---:|:---:|
| L1 逐句复述 | 听一句 → 暂停 → 口头复述 | 检测每一个词的识别率 | 30min | 每句 3 遍 |
| L2 段落听记 | 听 30-60 秒不暂停 → 记 3-5 个关键词 → 口头概述 | 信息抓取和重组 | 15min | 整段 2 遍 |
| L3 全文问答 | 听完整段 → 回答 3-5 个预设问题 | 真实对话理解 | 5min | 1 遍 |

**影子跟读放置：** 三层精听全部完成后，在最后 5 分钟（07:45-07:50）取当天音频的末段 60-90 秒做影子跟读——落后原声 0.5-1 词实时跟读，不暂停。周末影子跟读放在 HIMYM 精学的场景影子跟读环节。

铁律：3 遍还听不懂 → 立刻看 transcript → 标记 → 录 Anki。不追求 100% 听懂。80% + 20% 精准标记 > 100% 死磕。

## 泛听 vs 精听：两种完全不同的训练

| | 精听（07:00-07:50） | 泛听（早晨/午餐/晚间） |
|---|---|---|
| **目的** | 精准解码每一个词 | 大量输入，训练语流感 |
| **遇到生词** | 暂停 → 查 transcript → 标记 → 跟读 5 遍 → 录 Anki | 放过去。同一词出现 3+ 次且每次都引起注意 → 事后脑记，Anki 时花 30 秒查 |
| **听懂率** | 追求 80%+ 精准识别 | 50-60% 抓大意即可 |
| **设备** | 坐下，有 transcript | 不需要屏幕 |

铁律：泛听不暂停、不查词、不标记。唯一例外是同一生词在泛听中反复出现 3 次以上——事后用一个词记备忘录，下次 Anki 录入时查，绝不在泛听过程中打断自己。

## 精读 vs 泛读：完全对称的区分

| | 精读（17:00-17:30） | 泛读（17:30-17:50） |
|---|---|---|
| **目的** | 查生词、拆长难句、积累表达 | 堆阅读量，训练速读和猜词能力 |
| **查词** | 遇到生词立刻查 | 不查词，靠上下文猜，猜不出跳过 |
| **速度** | 慢，一句读不懂可反复读 | 快，不回读，不停留 |
| **理解率** | 90%+ | 60-70% 抓大意即可 |
| **产出** | 生词录 Anki，好句子记下用于写作 | 无产出，纯输入 |
| **材料难度** | 略高于当前水平 | 低于或等于当前水平 |

泛读训练的是「在不完美的理解中继续前进」——和多伦多真实场景中读租房合同、政府网站、菜单完全一致。

## 各模块材料来源与去重规则

### 泛听三段去重

晨间（06:30）、午餐（12:30）、晚间弹性（21:20）三段泛听：

| 时段 | 策略 | 原因 |
|------|------|------|
| 晨间 | **以新内容为主** | 唤醒耳朵 + 接触新输入 |
| 午餐 | **可重复晨间内容** | 低认知状态做巩固回顾 |
| 晚间 | **以舒适熟悉材料为主** | wind down，不设新挑战 |

晨间和精听的材料应该分离——不要让泛听剧透精听内容。

### 晚间口语② 材料来源与时间拆分

晚间口语②（19:00-19:40，总计 40 分钟）拆分为三个子活动：

| 子活动 | 时长 | 操作 | 材料 |
|--------|:---:|------|------|
| 🗣️ **自言自语** | 15min | 围绕话题连续说 3min → 回听 2min → 改进说 3min → 存档最佳 1min。话题=当天 ChatGPT 场景的英文总结或当天精听内容的个人复述。必须录音、必须回听 | **无外部材料**，纯输出 |
| 🎭 **跟读复述** | 15min | 取当天精听材料最后 2min 音频 → 影子跟读 3 遍（不暂停）→ 暂停后口头复述大意 1min。目标流畅度，非精确度 | **当天精听音频** |
| 🎲 **串讲训练** | 10min | 从当天精听/阅读中提取 3 个关键词 → 在 3min 内连成连贯即兴故事（英文）。不写稿、不准备。录音 | **当天精听/阅读中的关键词** |

三个子模块的材料不同：自言自语和串讲训练均无需外部材料（纯输出/即兴），跟读复述使用当天精听音频。

### ChatGPT 场景选择方法

Phase 1-2：严格按照计划文档 §3 每日计划中的场景执行，覆盖全部多伦多生存基础场景（咖啡店→问路→租房→银行→看病等）。Phase 3-4：由当周主题决定。和当天精听主题不强求对齐，但自然呼应更好。

### 写作主题演进

写作主题随阶段逐级升级：自我介绍 → 日记 → 实用邮件（租房/退货/银行）→ 观点短文（200-250 词）→ 听后写摘要。所有作文：写 → ChatGPT 改 → 抄写修改版 3 遍。

### 自测录音（周度回顾，非独立训练）

自测录音不是周日突然多出的一项训练——它是每日自言自语录音的「周度回顾」：一周 6 天的自言自语全部在录音，周日选取一段最新的，和上周日对比。听发音清晰度、卡顿减少、用词丰富度。听出进步本身就是最好的正反馈。

## HIMYM 三阶段精学法

> ⚠️ **关键区分：** 工作日晚间 20:40-21:10 的 30 分钟标注为「HIMYM/场景」——这是**轻松看剧**（英文字幕、不暂停、不分析）或场景模拟练习，不是精学。真正的三阶段精学全部在**周六下午 2 小时**完成。

每集 ~780 行英文台词，22 分钟。每集需要 **3 个周六** 完成三个阶段，阶段之间自然间隔一周（让大脑巩固）。

| 阶段 | 第几个周六 | 核心操作 | 时间 |
|------|:---:|---------|:---:|
| 阶段 1（输入） | 第 1 个 | 完整看 → 逐句跟读前 10min → 标 20 个实用表达 → 录 Anki → 再看一遍 | ~2h |
| 阶段 2（内化） | 第 2 个 | 无字幕看 → 逐段复述（40min）→ 挑 3 场景影子跟读（25min）→ 录音对比原声（15min）→ 用 20 个表达各造 2 个新句子（18min） | ~2h |
| 阶段 3（输出） | 第 3 个 | 无字幕看 → 全剧影子跟读（22min）→ 口头复述全剧情 → ChatGPT 角色扮演 → 写观后感 200 词（16min） | ~1.5h |

工作日 30 分钟的用法：当周在推某集时，可看一遍该集回顾（不暂停）或选一个场景自言自语模仿角色对话。其他周可改做场景模拟训练。

**「第 N 轮」= 第 N 个阶段。** 第 1 轮=阶段 1（输入），第 2 轮=阶段 2（内化），第 3 轮=阶段 3（输出）。避免使用「轮」这个模糊词汇，直接用「阶段」。

## 阿飞口语 31 天课程

⚠️ **纠音内容为中文讲解**（口型、舌位），不能作为英语听力材料。场景应用 mp3 为中文讲解+英文示范混合格式，同样不适合做精听。**仅用于发音训练。**

- Day 1-15：音标逐个击破（元音/辅音）
- Day 16-31：连读/吞读/重读/语调/弱读（恰好覆盖用户「听不出」的语音现象）
- Day 29-30：美剧精讲（课程自带的 TV 片段分析）

## 常见陷阱

1. **将中文讲解的发音课程用作英语听力输入** → 阿飞口语的纠音和场景应用都不能当精听材料
2. **为初学者推荐语速太快的播客** → 必须区分「播客本身不错」和「适合当前阶段」。All Ears English 虽然语速为母语自然速度，但内容为基础日常表达 + Apple 播客可降速 → 适合
3. **忽略 Apple 播客的调速和 transcript 功能** → Apple 播客支持 0.5x-2x 速 + 内置 transcript，调研播客时必须先核实这个
4. **默认学习时间是全天连续的** → 用户有正常工作，时间分散在四个时段
5. **忽略「认识但听不出」和「真生词」的复习策略差异** → 前者应该仅做听音辨义训练，后者需要完整学词义+拼写。Anki 用不同 Deck 分开
6. **给三层精听每层准备不同材料** → L1/L2/L3 是对同一段音频的三层递进用法，不要准备 3 个不同音频
7. **每个阶段重新写一份时间表** → 时间框架四个阶段完全通用，各阶段计划文档只列材料，不重复列时间
8. **自测录音当作孤立训练** → 它本质是每日自言自语录音的周度对比回顾，不是周日凭空新增的训练
9. **晨间泛听和精听使用同一材料** → 泛听变「剧透」，精听效果打折。两段应使用不同材料
10. **ChatGPT 对话场景需要自己编** → Phase 1-2 场景已预设在 §3 每日计划中（咖啡店、问路、租房…），覆盖多伦多全部生存场景，严格按计划执行即可
11. **忘记每周日更新豆包 System Prompt** → 每周日下午准备材料时，必须把当周 5 个场景追加到 System Prompt 末尾的 `This week's scenarios:` 列表。场景来源 = 计划文档 §3 当日「口语①」列。豆包语音模式不支持 System Prompt 时，每次对话开始用英文发一句话设定场景
12. **HIMYM 精学阶段塞到工作日 30 分钟里做** → 精学三阶段每阶段需 1.5-2h，工作日 30 分钟根本不够。精学固定周六下午，工作日只看剧或练场景
13. **泛听时停下来查词做笔记** → 泛听唯一目的是堆量，暂停会打碎语流感。如果真有个词反复出现 3+ 次，事后用一个词记备忘录即可，绝不在泛听过程中打断自己
14. **泛读时遇到生词就查词典** → 泛读训练的是在不完美理解中继续前进的能力。查词 = 把泛读变成了精读，等于浪费了当天的速读训练窗口
15. **计划文档过度注释和冗余** → 用户明确反馈：只保留「几点到几点做什么 + 用什么材料」，去掉讨论过程、教学比喻、重复注释和 blockquote 解说。文档是执行手册，不是教科书
16. **每个阶段重新列一份完整时间表** → 时间框架四个阶段完全通用，§3 只列每天的「内容」，不重复列时间
17. **给计划文档盲目追加 blockquote 注释** → 用户已明确反感过度的 `>` 注释框。修订文档时避免新增解释性 blockquote，能融入表格的融入表格，能删的删

## 计划文档维护约定

Colin 的完整学习计划文档位于 `~/Developer/GitHub/KnowledgeBase/English/12周多伦多生存英语学习计划.md`。该文档是本 skill 的「实例化」产物——skill 定义方法论和原则，文档承载具体的 12 周逐日内容。维护原则：

- **保持精简：** 文档是执行手册，只包含「时间 + 动作 + 材料」。不包含讨论过程、教学比喻、哲学解释
- **避免 blockquote 注释堆积：** 解释性内容融入表格备注或直接删除，不另起 `>` 块
- **关键字段加粗：** `**字段名**`
- **交叉引用用锚点链接：** `[§1.2](#1-2-xxx)`
- **避免破坏现有表格结构和 12 周逐日数据**
- **当冗余积累到需要大范围清理时：** 用 `write_file` 全量重写，不逐条 patch
