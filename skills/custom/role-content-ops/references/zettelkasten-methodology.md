# 卡片笔记写作法（Zettelkasten）方法论

> 所有笔记的最终沉淀载体为 Obsidian。不存在任何绕过 Obsidian 的归档路径。

## 核心理念
1. **自下而上**：不预设框架，从阅读与思考中自然涌现主题
2. **原子化**：每条笔记只承载一个独立想法（One Idea Per Note）
3. **网状链接**：笔记间通过语义化双链建立关联
4. **洞见驱动**：笔记不是摘录，是思考的结果

## 笔记类型体系

```
闪念笔记(Fleeting) → 快速提炼判断价值
文献笔记(Literature) → 深度加工提炼洞见
永久笔记(Permanent) → 聚合关联构建论证
项目笔记(Project) → 产出发布多平台适配
最终产出(Article/Video/Post)
```

### 闪念笔记
- 用户提供的灵感片段/关键词/未加工想法
- 低价值→ `Ideas/Fleeting/` + `#fleeting`；高价值→转化为文献/永久笔记
- 48小时内未处理则自动降级

### 文献笔记
- 从外部素材提取核心论点+关键数据+个人批注
- 必须包含：来源元数据（作者/链接/日期）+ 核心要点 + 我的批注
- 存入 `Sources/Literature/` + `#literature`

### 永久笔记（核心能力）
| 维度 | 标准 |
|------|------|
| 原子性 | 一个想法一条笔记 |
| 自足性 | 脱离原始上下文仍可独立理解 |
| 洞见性 | 包含提炼后的思考，不是复述 |
| 链接性 | 主动寻找与已有笔记的逻辑关联 |
| 标签克制 | ≤5个标签，对齐已有标签体系 |

### 项目笔记
- 围绕产出目标聚合相关永久笔记
- 临时性笔记，产出完成后归档
- 存入 `Projects/` + `#project`

## 双链设计哲学
- **语义化链接**：将链接融入句子，"这种思路与[[xxx]]的核心逻辑一致"
- **避免机械堆砌**：段落末尾 `相关链接: [[A]] [[B]] [[C]]` 是反面示例
- **三问检验**：读者点击后获得什么？在当前论证中扮演什么角色？是否有更自然的表述？

## 多平台内容适配
- **核心原则：** 内容同源、表达异构——核心观点不变，叙事结构按平台调整
- 小红书：emoji+数字+痛点钩子，分点口语化，5-8标签
- X Thread：首条钩子+核心结论，后续展开，末条CTA
- 微信公众号：场景代入+悬念+分段小标题+互动引导
- YouTube：关键词+悬念，简洁有力
